Você já deve ter feito códigos ou documentos no qual o nome do arquivo final era algo como: “arquivo final agora vai ultimo imprimir foi.docx”. Então você já deve ter percebido a importância de manter o versionamento de seus arquivos. Como explicado no tutorial introdutório o Git te oferece uma série de ferramentas que ajudam a fazer e manter o versionamento dos arquivos. Nesse tutorial vamos falar sobre os primeiros passos para a utilização do git com o TortoiseGit. Vamos descobrir como criar um repositório de trabalho para depois poder começar a versionar seu arquivos.

Para acompanhar esse tutorial você deve ter instalado o TortoiseGit. Caso você ainda não tenha instalado o TortoiseGit veja o tutorial para instalar o TortoiseGit .

Passo 1: criando um repositório de trabalho

O diretório de trabalho do git pode ser definido como um mini-sistema de arquivos aonde as ferramentas do git irão agir. Qualquer diretório do windows pode se tornar um diretório de trabalho. Para isso basta inicializar nesse repositório um novo repositório de trabalho. Cada repositório de trabalho inicializado possui seu próprio histórico de mudanças, para montar um usando o TortoiseGit basta criar uma nova pasta no windows clicar com o botão direito em cima da pasta e no menu de contexto do explorer escolher a opção git Create repository here como mostrado na figura 1

Figura 1. Criando repositório de trabalho

Após clicar na opção de criar um repositório da figura 1 vai aparecer uma mensagem de aviso (figura 2). Nessa mensagem de aviso vai pedir se você quer realmente montar um novo repositório de trabalho ou se você quer montar um repositório descoberto (em ingles bare ). O uso desse tipo de repositório é assunto pro próximo tutorial, mas ele basicamente serve para o usuário criar um repositório compartilhado. (e.g., um repositório que várias pessoas podem enviar seus históricos). Por enquanto não marque a opção “make it bare” e clique no botão ok para criar o repositório de trabalho.

Figura 2. Mensagem de aviso do TortoiseGit do tipo de repositório a ser criado.

Se tudo ocorreu como o planejado, dentro da pasta vai conter uma pasta oculta com o nome “.git” . Ao clicar com o botão direito dentro da pasta de trabalho, irá abrir o menu de contexto contendo agora as opções:

  • Git Sync
  • Git commit
  • tutoiseGit

Essas opções podem ser vistas na figura 3

Figura 3. Menu de contexto do explorer contendo as ferramentas do TortoiseGit habilitadas.

Próximo post: como utilizar o comando commit

Agora você já possui um diretório no qual pode-se utilizar as ferramentas de versionamento do git. Agora aprenda a utilizar a ferramenta de commit do git . A ferramenta de commit é a ferramenta responsável por gravar o estado atual dos arquivos. Ela é a ferramenta mais básica do git. Caso você tenha alguma dúvida poste nos comentários.
