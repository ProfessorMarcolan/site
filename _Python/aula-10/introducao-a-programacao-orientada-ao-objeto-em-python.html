Nesse poste introduziremos o paradigma da orientação ao objeto em Python. Aqui no site já abordamos tópicos sobre o paradigma da programação estruturada . Tal paradigma geralmente é o primeiro ensinado nos cursos de engenharia e da ciência da computação porque ele é muito eficaz na criação e no estudo de algoritmos (e.g. algoritmo de ordenação , algoritmo a*). Porém, quando você começa a criar softwares (programas com mais quantidades de linhas) você começa a precisar a desenvolver utilizando o paradigma da orientação ao objeto. Vamos entender o porque disso!

Aonde mora o problema

Um algoritmo de ordenação tem entre 10 a 50 linhas de códigos. O que é bem fácil para só um programador escrever. Mas um algoritmo de ordenação não é um programa é apenas uma pequena parte dele. Imagine por um momento que você tem tantas linhas de códigos quanto a moça na foto 1. São muitas linhas de código para apenas uma pessoa escrever. Para criar programas grandes como esse você precisará de uma serie de recursos e praticas. Tais recursos você encontrara no paradigma da orientação ao objeto. E práticas interessantes para o desenvolvimento são as do acrônimo SOLID . Mas antes de progredir o que é um paradigma mesmo ?

O que é um paradigma

Existem muitas formas de se escrever um texto com uma linguagem. Em resumo uma linguagem tem apenas palavras e regras de como usá-las . O uso dessas palavras e regras fica a merce do escritor. Então, por exemplo, com português você pode escrever textos científicos assim como cronicas. O cientifico será um texto descritivo e a poesia será um texto extremamente subjetivo. Os dois textos serão totalmente diferentes mas estarão usando a mesma linguagem. Então, um paradigma pode ser definido com uma forma de como escrever o código. E para escrever tal código será lhe dado um conjunto de ferramentas.

O que é o paradigma da orientação ao objeto em Python

Um paradigma em ciências da computação é geralmente um método de categorizar os recursos que uma linguagem te oferece. Então, por exemplo, o Python é uma linguagem que tem o paradigma da orientação ao objeto. Para ela ser classificada como tal ela deve te oferecer 3 recursos : herança, polimorfismo, encapsulamento. Esses três recursos são usados para ORGANIZAR os códigos em classes.

Quando você programa utilizando o paradigma da orientação ao objeto em Python todo seu esforço é em como usar os três recursos disponíveis para organizar seu código. Programar utilizando o paradigma da orientação ao objeto é diferente de programar utilizando o paradigma procedural. No paradigma procedural você está utilizando os recursos do paradigma para concatenar os diferentes procedimentos. Geralmente quebrando o objetivo geral (e.g. ir do ponto A para o ponto B) em pequenos passos lógicos. No paradigma da orientação ao objeto seu esforço está em organizar todas as ações e dados em classes .

Transição do paradigma procedural para o paradigma de orientação ao objeto

Geralmente o programador começa tendo um código estruturado no paradigma procedural e/ou pouco organizado. Geralmente o estudo começa no intendimento de uma nova biblioteca/Framework já desenvolvida. O entendimento é feito através da implementação de algumas funções utilizando as bibliotecas escolhidas. O recomendado é até mesmo fazer essas funções junto com os testes automáticos dessas funções. A partir desse primeiro contato o programador começa lentamente organizar o código em classes. Dificilmente você iniciará a criação de um software sabendo todas as nuances desses software. Você poderá ter um chute inicial. Porém, conforme seu entendimento vai aumentando sobre o problema e sobre o seu software ele vai se modificando. Esse tópico é discutido no post valores agile.
